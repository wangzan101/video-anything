# 黄金样例:口播稿改写(rewrite)

这份样例给「口播稿改写」playbook(见 `SKILL.md` 第 4 步)一个具体的质量锚点:
同一段输入素材,改写成 3 种预设风格分别长什么样、差异体现在哪里、又如何都不越过
"不编造原视频没有的事实/数据"这条硬约束。输入素材沿用 `key-points.md` 那段
「反拖延技巧」的 `info.json` + `transcript.md`(与 `scripts/fetch.sh` +
`scripts/transcribe.py` 实际产出格式一致),方便对照两份黄金样例、体会
"提炼要点"与"改写口播稿"这两个 playbook 产出的本质区别。

## 输入 1:`info.json`(节选)

```json
{
  "id": "7123456789012345678",
  "title": "自由职业三年,我总结的4个反拖延技巧",
  "uploader": "阿凯的效率日记",
  "duration": 250,
  "description": "别再靠意志力硬扛了,这四个方法我自己用了快两年,亲测有效。",
  "webpage_url": "https://www.douyin.com/video/7123456789012345678",
  "tags": ["效率", "自我提升", "拖延症"]
}
```

用户没有指定风格,于是分别按 3 种预设风格各写一版,展示给用户挑选。三版
共用同一批事实(自由职业三年、4 个技巧、亲测快两年、"完成率八成以上"
"效率能差两倍不止"这两处原文明确给出的具体说法),不额外添加原文没有的
数字或细节;三版在人称、结构、节奏、开头结尾上刻意拉开差异。

## 期望产出 2:知识拆解风格

```markdown
# 自由职业三年,我总结的4个反拖延技巧

## 风格:知识拆解

结论先说:治好拖延症的关键思路,不是靠意志力硬扛,而是降低"开始"这个
动作的门槛。下面是分享者自由职业三年、亲测快两年的 4 个具体方法。

**方法一:两分钟启动法**
任何任务先规定自己只做两分钟,时间到允许停下。多数情况下,两分钟之后
你已经进入状态,根本不会想停。

**方法二:手机物理隔离**
把手机放到另一个房间,而不是仅仅调静音——静音状态下手机仍在身边,大脑
还是会一直惦记它;只有物理隔离才是真正有效的做法,分享者提到亲测效率
能差两倍不止。

**方法三:今日三件事**
每天早上花两分钟,写下当天最重要的三件事,不多不少正好三件。写多了等于
没写,因为大脑没法给超过三个目标排优先级。坚持下来,完成率能到八成以上。

**方法四:及格线交付**
很多拖延不是懒,是完美主义在作祟——怕做不好,干脆不做。解法是先交出一个
"六十分"的版本,东西摆在眼前,反而会忍不住想去改进它。

如果时间有限,优先做方法一:两分钟启动法,成本最低,今天就能验证效果。

> 适用平台/时长建议:适合图文长文或短视频知识类口播稿,时长视展开程度而定,通常控制在 1–3 分钟内讲完 4 个方法。
```

## 期望产出 3:播客口播风格

```markdown
# 自由职业三年,我总结的4个反拖延技巧

## 风格:播客口播

欢迎回来,今天想跟大家聊一个特别实际的话题——拖延。

先说个背景。今天聊的这位分享者是自由职业者,做了三年,他说前两年基本
每天都在跟自己的意志力较劲,后来才想明白,这压根不是意志力的问题。

那到底是什么问题呢?他的说法是——难的不是任务本身,难的是"开始"这个
动作。所以他给自己总结了四个方法,自己用了快两年,说是真的有效,把他
从一个重度拖延症里救回来了。

先说第一个,叫"两分钟启动法"。听着挺简单,其实思路挺妙的:任何任务,
先规定自己只做两分钟,两分钟一到,你可以选择停下来。神奇的地方在于,
大部分时候两分钟之后你已经进入状态了,反而不想停了。

聊完这个,再说说手机的问题。很多人可能会想,那我把手机静音不就行了?
他说不行,静音没用,因为手机还在你手边,大脑其实一直在惦记它。真正
有效的是物理隔离——放到另一个房间去,他说自己亲测过,效率能差两倍不止。

第三个方法,叫"今日三件事"。每天早上花两分钟,写下今天最重要的三件
事——注意,是三件,不是十件。写多了等于没写,因为咱们的大脑没办法同时
给超过三个东西排优先级。他说自己只要坚持写这三件事,那天的完成率基本
能到八成以上。

最后一个,也是他觉得最反直觉的一个——允许自己不完美地完成。很多时候你
拖延,不是懒,是完美主义在作祟,怕做不好干脆就不做了。他给这个起了个
名字,叫"及格线交付",先交出一个六十分的版本。东西一旦摆在你眼前,你
自己都会忍不住想去改进它。

如果这四个方法你只能记住一个,他建议记第一个,两分钟启动法,因为它
成本最低,今天就能试。

最后他留了一句话,我觉得挺打动人的——拖延的反义词不是自律,是开始。

好,这期就聊到这,我们下期见。

> 适用平台/时长建议:适合播客或长视频口播,时长可以比原视频(约 4 分钟)更长,重点在营造对话感和过渡的自然度。
```

## 为什么这 3 版都"合格"

- **都基于同一批事实,没有一处编造**:三版共同引用的具体数字("完成率
  八成以上""效率能差两倍不止""自由职业三年""亲测快两年")都能在
  `transcript.md` 里找到原句依据,没有任何一版为了风格效果而添加原文
  没有的数据或细节。
- **风格差异体现在人称、结构、节奏,而不是内容取舍**:小红书版用第一
  人称+钩子开头+emoji+互动收尾;知识拆解版结论先行、分点带小标题、
  语气克制;播客版用大量过渡句("先说个背景""聊完这个,再说说……")
  制造对话感,篇幅明显更长。三版都完整覆盖了 4 个技巧和金句,没有因
  为风格不同而漏掉某个技巧。
- **都是重组表达,不是照抄原句**:例如"物理隔离才是真的有效,这个我
  亲测过,效率能差两倍不止"这句原文,三版分别用种草式短句、说明文式
  从句、播客式转述句改写,措辞都变了,但事实没有变形或夸大。
- **每版都标了风格 + 附了适用平台/时长建议**,与 playbook 的输出结构
  一一对应,方便直接对照校准。
